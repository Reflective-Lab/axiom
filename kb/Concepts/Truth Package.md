# Truth Package

A Truth Package is the typed, reproducible bundle Axiom produces from a
structured JTBD. `.truths` is one auditable projection inside the package, not
the primary human input.

## Release Scope Through v0.15

The release package spine is deliberately narrow:

```text
JtbdInput
  -> JtbdDocument
  -> generated .truths projection
  -> optional TruthProjectionOverlay
  -> IntentPacket
  -> VerifierSpec
  -> ReplayProfile
  -> LineageMap
  -> AxiomRunObservation
  -> AxiomRunReport
  -> LearningEpisode
  -> CalibrationRecord candidates
```

Policy synthesis and broad domain generation remain future work. The release
surface is the deterministic package, the app-neutral observation/report
contract, adapter receipts, lineage audit, and reviewed decoder calibration.

## Manifest Fields

| Field | Purpose |
|---|---|
| `package_id` | Deterministic package identity derived from the canonical JTBD clauses. |
| `truth_version` | Package-local truth version; v0.10 emits `v1`. |
| `source_jtbd` | Canonical `JtbdDocument` with stable clause IDs and fingerprints. |
| `generated_truths` | Auditable `.truths` projection generated by deterministic rules. |
| `artifacts` | Grouped generated artifact summaries for scenarios, predicates, policy requirements, evidence expectations, simulation cases, invariant expectations, and accepted calibration suggestions. |
| `intent_packet` | Runtime contract consumed by Organism. Axiom sets a deterministic packet ID for package reproducibility. |
| `proof_obligations` | Evidence and failure-mode duties derived from source clauses. |
| `verifier_spec` | Expected stop reasons, required evidence, forbidden actions, and satisfaction conditions. |
| `replay_profile` | Deterministic replay metadata for package inputs. |
| `lineage` | Artifact-to-clause custody map with closure validation. |

## Run Report Shape

`AxiomRunReport` is the app-neutral verifier record. It carries:

- package ID, truth version, intent packet ID, and source clause IDs;
- the `VerifierSpec` used for judgment;
- an `AxiomRunVerdict`: `Satisfied`, `Blocked`, `Exhausted`, or `Invalid`;
- the observed stop reason, using a Converge-compatible shape without importing
  Converge internals;
- promoted fact summaries with source clause IDs, evidence refs, trace links,
  and optional observed promotion authority records from Converge;
- the run integrity proof summary and replay notes.
- optional `AxiomRunStageRecord` entries for jobs with more than one Converge
  boundary.

Callers normalize run output into `AxiomRunObservation` and call
`AxiomRunReport::verify(&package, observation)`. App-specific and runtime
adapters remain caller-owned; Axiom owns the normalized observation, report, and
verdict semantics.

The first staged fixture is [[Marquee/Round-Driven Formation Design]], where
a design huddle Formation converges before a selected work Formation runs. The
top-level report carries the overall verdict; stages preserve each Formation's
stop reason, promoted facts, trace links, and integrity proof.

## Observation Adapter Receipts

`ObservationAdapterReceipt` is the app-neutral audit envelope for adapters that
map raw app or runtime transcripts into `AxiomRunObservation`.

The app owns the raw transcript and mapping logic. Axiom owns the receipt
schema: adapter id/version, success or rejection status, source transcript ref
and hash, package id, truth version, domain hint, optional observation hash,
mapped fact ids, mapped clause ids, warnings, errors, and replay notes. The
receipt id is content-derived from the full receipt input.

Receipts are backlink-oriented. They do not store raw app transcript bodies and
they do not grant authority. Helm can render or journal them while Axiom still
verifies the resulting observation against the package.

## Provenance Seed Facts

Axiom declares `AxiomTruth` and `AXIOM_PROVENANCE` as its
`converge_pack::ProvenanceSource`. Truth Package seed proposals use
`AXIOM_PROVENANCE.proposed_fact(...)` and carry a typed
`TruthPackageSeedPayload` with:

- package ID and truth version;
- clause ID, clause kind, and clause fingerprint;
- canonical clause text.

These proposals target `ContextKey::Seeds`. Converge still owns promotion; when
a seed proposal is promoted, the promoted fact keeps the typed Axiom payload and
Converge attaches the promotion record, evidence refs, trace link, and integrity
proof.

## Projection Overlays

Human edits do not mutate `generated_truths`. They live in a
`TruthProjectionOverlay` that names:

- target package ID;
- target truth version;
- projection version;
- edited `.truths` text;
- human reason;
- source clause IDs the override serves.

Applying an overlay returns a `TruthProjectionVersion`. The base package remains
regenerable and idempotent. Overlays are rejected if they target the wrong
package, target a stale truth version, reference unknown clauses, omit a reason,
or contain `.truths` text that no longer parses.

## Decoder Calibration Suggestions

v0.13 adds decoder calibration as an auditable enrichment layer. An audited
`AxiomRunReport` plus `FactLineageAudit` can become a `LearningEpisode`; the
episode can produce proposed `CalibrationRecord` rows; accepted rows can be
applied back to a regenerated package with `apply_decoder_calibration(...)`.

The enriched package receives `ArtifactKind::CalibrationSuggestion` artifacts.
Each suggestion cites both the JTBD clause it serves and the calibration record
that influenced it through artifact lineage. Calibration does not change the
source JTBD, generated `.truths`, `IntentPacket`, authority model, or Formation
selection.

## Determinism Rules

- The same semantic JTBD input must regenerate the same package.
- Reordering evidence or failure-mode clauses must not change package identity.
- Generated `IntentPacket` IDs are deterministic inside a Truth Package.
- Generated `.truths` projections use a deterministic expiry anchor. When a
  JTBD declares a `TimeBudget`, Axiom carries that budget in package/runtime
  metadata so time exhaustion can be an honest verifier outcome.
- Every generated artifact names its source clause IDs, decoder rule, decoder
  version, and input fingerprints.
- Every calibration-influenced artifact names the accepted calibration record
  and still passes normal lineage closure against the source JTBD.
- Every source clause must be used, explicitly deferred, or explicitly rejected.
- Overlays are separate projection versions; applying one must not change the
  generated package.

## Boundaries

Axiom produces the package. Organism consumes the `IntentPacket` and reasons
with formations. Converge recomputes authority and promotes facts. Helm hosts
WASM plugins when executable artifacts need a sandbox.
