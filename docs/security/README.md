# Security and Compliance

This directory collects the repository-level security and compliance material
that enterprise reviewers typically ask for during initial diligence.

## Included Documents

- `SECURITY_OVERVIEW.md` - current security posture and control summary
- `DATA_HANDLING_DECLARATION.md` - what this project is designed to handle, and what it is not
- `COMPLIANCE_READINESS.md` - present-state declarations and recommended path to common frameworks
- `THREAT_MODEL.md` - major risks, assumptions, and mitigations

## Scope

These documents describe the open source project and the reference runtime in
this repository. They do not by themselves constitute a certification,
attestation, data processing agreement, or regulatory opinion.

Deployers remain responsible for:

- environment hardening
- identity provider configuration
- encryption key management
- retention and deletion settings
- vendor selection and subprocessors
- customer-specific legal and regulatory obligations
