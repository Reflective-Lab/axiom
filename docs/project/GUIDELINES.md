# Project Guidelines

## Plans

- One file per plan, named descriptively (e.g., `llm-provider-abstraction.md`)
- Start with a problem statement before proposing a solution
- Include scope, dependencies, and affected repos
- Mark status at the top: `draft`, `active`, `completed`, `abandoned`

## Roadmap

- Organize by quarter or phase
- Keep entries concise — link to plans for details
- Update regularly as priorities shift

## Milestones

- Define clear success criteria
- Track target dates and actual completion
- Link to relevant plans and repos

## General

- Keep documents short and actionable
- Prefer updating existing docs over creating new ones
- Use plain markdown, no special tooling required
